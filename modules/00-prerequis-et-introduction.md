# Module 00 — Prérequis et introduction

> **Objectif** : comprendre ce que ce cours couvre vraiment, pourquoi l'algorithmie compte dans une carrière JS fullstack, et comment progresser sans se perdre dans le théorique pur.

> **Difficulté** : ⭐ — accessible dès que tu sais lire et écrire du JavaScript.

---

## 1. Ce cours n'est pas un concours de puzzles

L'algorithmie utile à un ingénieur n'est pas seulement une affaire d'entretiens techniques. Tu la recroises dans des sujets concrets : pagination, déduplication, recherche, autosuggestion, règles métier, ranking, cache, file d'attente, ordonnancement, synchronisation.

Le but ici est simple : **développer des réflexes de résolution**.

## 2. Les trois questions à se poser devant un problème

1. **Quelles sont les entrées et les contraintes ?** Taille des données, ordre, doublons, mises à jour.
2. **Quelle structure de données aide le plus ?** Tableau, `Map`, `Set`, heap, arbre, graphe.
3. **Quel coût est acceptable ?** Temps, mémoire, simplicité, maintenabilité.

## 3. La progression du cours

- d'abord les structures linéaires et le coût ;
- ensuite les décompositions (`binary search`, récursion, tri) ;
- puis les structures hiérarchiques et relationnelles (arbres, graphes) ;
- enfin les patterns d'optimisation et les cas produit.

## 4. Ce que tu dois retenir

- Une solution “qui marche” n'est qu'un début.
- Une bonne solution est **correcte, lisible et proportionnée** à la taille du problème.
- En ingénierie, l'algorithme gagnant est souvent le plus simple qui respecte les contraintes.

## 5. Parcours recommandé

Lis un module, fais le lab associé, puis fais le quiz. Si un pattern reste flou, reviens-y plus tard après deux ou trois autres problèmes.
